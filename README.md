# Novel Object Recognition — Analysis Workflow

This repository contains the full analysis pipeline for a Novel Object Recognition (NOR) task, from raw data cleaning to final publication-ready figures. It is structured as an RProject and covers data wrangling, descriptive statistics, mixed-effects modelling, and consistent ggplot2 visualisations.

> **A small simulated dummy dataset is included in `raw_data/`** so you can run the workflow out of the box and explore the outputs. When you are ready to use your own data, replace the dummy file with your own CSV — just make sure it follows the column structure described in [Input data format](#input-data-format).

## Setup

1. Clone or download this repository.
2. Create a R project (`.Rproj` file in RStudio) in the repository folder. **All file paths in the scripts are relative to the project root**, so the RProject must be open for them to work correctly.
3. Install required packages (see `Functions/load_packages.R`). Key dependencies are:

   - `tidyverse` — data wrangling and ggplot2
   - `lme4` — linear mixed-effects models (`lmer`)
   - `glmmTMB` — generalised mixed models with beta family for the discrimination index
   - `DHARMa` — residual diagnostics for `glmmTMB` models
   - `emmeans` — estimated marginal means and post-hoc comparisons
   - `car` — `Anova()` for Type II/III tests
   - `rlang` — tidy evaluation used in the plotting functions

## Input data format

Place your raw data file at `raw_data/test_dataset.csv`. The script expects the following columns:

| Column | Type | Description |
|---|---|---|
| `Animal` | factor | Unique animal identifier |
| `Sex` | factor | `"F"` or `"M"` |
| `Treatment` | factor | `"1_min"` or `"3_min"` (inter-trial interval) |
| `Session` | factor | `"TR"` (training) or `"TS"` (test) |
| `Type_object` | factor | `"Fam"` (familiar) or `"Nov"` (novel) |
| `Object` | character | Object identity label |
| `Location` | character | Object location label |
| `Exploration` | numeric | Exploration time in seconds |

> The file is read with `fileEncoding = "Latin1"` to handle special characters. Change this in `cleaning_analyzing.R` if your system uses a different encoding.

## Step 1 — Cleaning, analysis and saving: `Scripts/cleaning_analyzing.R`

Run this script first. It covers the full analysis pipeline and saves all outputs needed by `final_plots.R`.

### What it does

**1. Data cleaning**
- Reads the raw CSV, drops rows with `NA` values, and enforces the correct factor level order for `Treatment` (`"1_min"` before `"3_min"`).
- Splits the data by session into `TS_data` (test) and `TR_data` (training).

**2. Discrimination index (DI) calculation**
- Reshapes the data from long to wide format (one row per animal per session) using `tidyr::pivot_wider()`.
- Computes three new columns:
  - `Total` — total exploration time (`Nov + Fam`)
  - `beta_di` — proportion of time spent exploring the novel object (`Nov / Total`), used as the model outcome (bounded 0–1, suitable for beta regression)
  - `DI` — the classic discrimination index as a percentage (`(Nov − Fam) / Total × 100`)

**3. Descriptive statistics**
- Uses the `descriptive()` function (see `Functions/statistics_functions.R`) to compute n, mean, SD, SEM, min, max, and variance, grouped by the relevant factors.
- Computed for both sessions (TS and TR) and all three outcome variables.

**4. Exploratory plots**
- Quick `ggplot2` plots for visual inspection of the data before modelling. These are not saved — they are only for exploration during the analysis.

**5. Statistical models**
Six models are fitted, three per session:

| Model | Session | Outcome | Model type |
|---|---|---|---|
| `model01` | TS | Exploration time by object | `lmer`: `Exploration ~ Treatment * Type_object + (1\|Sex/Animal)` |
| `model02` | TS | Discrimination index | `glmmTMB`: beta regression with logit link, `(1\|Sex)` |
| `model03` | TS | Total exploration | `lmer`: `Total ~ Treatment + (1\|Sex)` |
| `model04` | TR | Exploration time by object | `lmer`: `Exploration ~ Treatment * Type_object + (1\|Animal)` |
| `model05` | TR | Discrimination index | `glmmTMB`: beta regression, no random effect |
| `model06` | TR | Total exploration | `lm`: `Total ~ Treatment` |

For each model, the script runs:
- **Assumption checks** via `assumptions()` — plots residuals vs. predicted and a QQ-plot, returns Shapiro–Wilk test result. For `glmmTMB` models, `DHARMa::simulateResiduals()` is used instead.
- **ANOVA table** via `car::Anova()`
- **Post-hoc comparisons** via `emmeans()`

**6. Saving outputs**
All outputs are saved as `.rds` files so `final_plots.R` can load them directly without re-running the analysis:

- `data/` — cleaned dataframes and descriptive statistics
- `results/` — one `.rds` list per model containing the model object, assumption checks, ANOVA table, and emmeans

## Step 2 — Final figures: `Scripts/final_plots.R`

Run this script after `cleaning_analyzing.R` has been run and all `.rds` files exist in `data/`.

### What it does

Loads the saved data and descriptives, then calls `plot_behavior()` to produce three plot types for each session (TS and TR), saving each as a PDF in `figures/`.

### The `plot_behavior()` function

All plots are generated through a single unified function defined in `Functions/plotting_functions.R`:

```r
plot_behavior(
  type,        # "exploration", "DI", or "total"
  datos,       # the dataframe to plot
  desc,        # descriptive statistics dataframe (for mean ± SEM overlay)
  y_var,       # for type = "total" only: unquoted column name for the y axis
  y_limits,    # optional numeric vector c(min, max) passed to coord_cartesian()
  colors,      # named color vector; defaults to treatment_colors from themes.R
  params       # list of graphical parameters; defaults to plot_params
)
```

- `"exploration"` — paired jitter + line plot showing exploration time for familiar and novel objects, with mean ± SEM overlay. X axis shows the interaction of `Type_object × Treatment`.

### Customising the look

- **Colors** — edit `treatment_palette` in `Functions/themes.R`. Keys must match the `Treatment` factor levels exactly.
- **Point size, jitter width, line width, error bar width** — edit `plot_params` at the top of `Functions/plotting_functions.R`.
- **Overall theme** (font sizes, margins, axis lines) — edit `custom_theme()` in `Functions/themes.R`.
- **Y axis limits** — pass `y_limits = c(min, max)` to `plot_behavior()`.

## Functions reference

### `Functions/statistics_functions.R`

**`descriptive(data, grups, variable)`**
Computes summary statistics for a numeric variable grouped by one or more columns.
- `data` — a dataframe
- `grups` — character vector of grouping column names, e.g. `c("Treatment", "Type_object")`
- `variable` — character string naming the response variable, e.g. `"Exploration"`
- Returns a dataframe with columns: `n`, `avg`, `SE` (SD), `min`, `max`, `var`, `sem`

**`assumptions(model)`**
Visual and statistical assumption checks for `lmer` and `lm` models.
- Plots residuals vs. predicted (homoscedasticity) and a QQ-plot (normality) side by side.
- Returns the result of `shapiro.test()` on Pearson residuals.
- For `glmmTMB` models, use `DHARMa::simulateResiduals()` instead (already done in `cleaning_analyzing.R`).

### `Functions/themes.R`

**`custom_theme(show_x_text = FALSE)`**
A `ggplot2` theme function. Pass `show_x_text = TRUE` to display x-axis tick labels (used automatically by `plot_behavior()` for interaction plots).

**`treatment_palette`**
Named character vector mapping treatment levels to hex colors. Edit here to change colors globally across all plots.

**`get_treatment_colors(data, var = "Treatment")`**
Subsets `treatment_palette` to only the levels actually present in `data`, so unused levels never appear in the legend.
